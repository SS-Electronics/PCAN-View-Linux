Detailed Description

Background worker threads, global state, and connect/disconnect logic.

Defines the application singletons g_app and g_gui and their lifecycle (app_state_init / app_state_cleanup). Hosts the three worker threads — thread_rx, thread_tx, and thread_stats — which communicate results back to the GTK main thread via gdk_threads_add_idle(). Also implements the in-memory trace capture (trace_record) and the app_do_connect / app_do_disconnect lifecycle that starts and joins the threads.
